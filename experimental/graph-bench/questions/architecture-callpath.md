# Task: brief an exported API's call path

I'm onboarding to this TypeScript project. Pick one commonly used exported API (a public function, a class method, or an entry point) and walk me through how a call flows from the public entry point down to where the real work actually happens (the database call, the I/O, the core algorithm).

Name the key files and symbols along the way and how each connects to the next. Read symbol names, signatures, and how declarations relate the way you would skimming unfamiliar code; you do not need to open every file. Keep it a tight briefing that reaches the real work, not an exhaustive trace.
